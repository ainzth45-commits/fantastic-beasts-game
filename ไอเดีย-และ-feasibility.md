# เกมที่ 2 — "ทามาก็อตจิทีม" (ชื่อชั่วคราว) · โน้ตไอเดีย + Feasibility

> สถานะ: **กำลัง brainstorm** — ยังไม่ล็อกดีไซน์ · เอกสารนี้ไว้จดไอเดีย + ประเมินว่าสร้างได้จริงไหม
> อัปเดตล่าสุด: 2026-07-11

## แก่นไอเดีย (จากเจ้านาย)

สัตว์วิเศษการ์ตูน 1 ตัว เริ่มจาก**ไข่** แล้วค่อยๆ โตตาม**ยอดขายจริง**ที่พนักงานทั้งทีมส่งเข้ามา
เหมือนพนักงานทุกคนช่วยกัน "เลี้ยง/ฟูมฟัก" ตัวเดียวกัน → พอโตเต็มวัย = ปลดล็อก **รางวัลจริงในชีวิต**

- **แสดงบนจอ TV** (ไม่ใช่ iPad เหมือนเกมที่1) — เป็น dashboard มีชีวิตในออฟฟิศ
- **ผูกกับ Supabase** — รับยอดขายที่พนักงานส่งผ่าน CRM ของตัวเอง แบบ realtime
- **สัตว์ต้องขยับ "เหมือนมีชีวิต"** ไม่ใช่ภาพนิ่ง
- **Codex ช่วยออกแบบ** (งานศิลป์/ตัวละคร)

## Core loop (ร่าง)

```
พนักงานปิดการขาย → บันทึกใน CRM → ยอดเข้า Supabase
      → จอ TV รับ realtime → สัตว์กินยอด/มีปฏิกิริยา (ประกาย, เด้งดีใจ)
      → ยอดสะสมข้าม threshold → สัตว์เลื่อนขั้น (ไข่ → ฟัก → เด็ก → โต → เต็มวัย)
      → โตเต็มวัย → ปลดล็อกรางวัลจริง 🎉
```

## Feasibility — ทำได้จริงไหม (ประเมินตามความเสี่ยง)

| ส่วน | ความเสี่ยง | เหตุผล |
|---|---|---|
| Supabase realtime รับยอด → ขึ้นจอสด | 🟢 ต่ำ | Supabase Realtime subscription เป็นฟีเจอร์มาตรฐาน |
| จอ TV web app | 🟢 ต่ำ | มี CRM-Dashboard-on-TV เป็นแนวอยู่แล้ว, React เดิมใช้ซ้ำได้ |
| logic ไข่→โต ตามยอดสะสม + ปลดรางวัล | 🟢 ต่ำมาก | แค่ threshold ข้ามขั้น |
| **สัตว์ขยับเหมือนมีชีวิต** | 🟡 กลาง (จุดเสี่ยงหลัก #1) | ขึ้นกับระดับความสมจริงที่เลือก — ดูตัวเลือกด้านล่าง |
| ~~CRM ปัจจุบัน → Supabase~~ | ✅ **เช็กแล้ว ไม่เสี่ยง** | ยอดเข้า Supabase อยู่แล้ว — ดูรายละเอียดด้านล่าง |
| ความสม่ำเสมอของตัวละครข้ามขั้นโต (AI gen) | 🟡 กลาง | AI gen รักษา "หน้าตาตัวเดิม" ข้ามขั้นยาก — ต้องคุม reference ให้ดี |

**สรุป: สร้างได้จริง** — แต่ควรทำ POC พิสูจน์ 2 จุดเสี่ยงก่อนลุยเต็ม

## ตัวเลือกทำสัตว์ให้ "มีชีวิต" (เรียงตามความคุ้ม/เสี่ยง)

1. **เฟรมภาพ 2-3 ท่า + procedural motion (แนะนำ)** 🟢
   Codex ปั้นภาพนิ่งไม่กี่ท่า (หายใจ/กะพริบ/กระดิก) ต่อขั้น → โค้ด (CSS/canvas) วนเฟรม + ซ้อนการขยับ (เด้ง โยก squash-stretch ประกาย particle ตอนยอดเข้า) → รู้สึกมีชีวิตโดยไม่พึ่งเทคหนัก
2. **Rive / Lottie (2D rigged)** 🟡
   ขยับลื่นกว่า แต่ต้อง rig — Codex ทำ rig ไม่ค่อยไหว, เพิ่ม tooling
3. **คลิปวิดีโอ/GIF loop ต่อขั้น (AI gen)** 🟡
   AI สร้างคลิปสั้นวนต่อขั้น — ได้ลื่น แต่ไฟล์ใหญ่ + คุมคุณภาพยาก
4. **3D (Three.js + .glb)** 🔴
   สมจริงสุด แต่ pipeline ศิลป์หนัก Codex ปั้น 3D ไม่ไหว — เก็บไว้อนาคต

- [x] **จบเดือนยังโตไม่เต็มวัย → เลี้ยงต่อข้ามเดือนได้** ไม่ตาย/ไม่รีเซ็ต แต่**อัตราโตช้าลง** (ค่าปรับความล่าช้า)
- [x] **ระบบอารมณ์ (mood)** — หัวใจของเกม:
  - อารมณ์ดี → โตเร็ว (ตัวคูณ growth สูง) · เศร้า/เหงา → โตช้า
  - อารมณ์ขึ้นกับสถานการณ์ยอดขายจริง ณ ตอนนั้น เช่น
    - **ความถี่ยอด**: ยอดเข้าติดๆ กัน = ร่าเริง · ห่างหายนานไม่มียอด = เหงา
    - **ขนาดยอด**: ดีลใหญ่ = ดีใจสุดๆ (ปฏิกิริยาพิเศษ) · ยอดเล็กกระปริดกระปรอย = เฉยๆ
    - **ภาพรวมวัน**: ยอดวันนั้นน้อย = เศร้า
  - อารมณ์แสดงออกทาง**ท่าทาง/แอนิเมชัน** — ทีมเห็นจอแล้วรู้ทันทีว่าน้องรู้สึกยังไง = แรงกระตุ้นให้รีบหายอดมาป้อน
