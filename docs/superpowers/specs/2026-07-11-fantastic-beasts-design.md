## 1. ภาพรวม

สัตว์วิเศษการ์ตูน 1 ตัว เริ่มจากไข่ โตขึ้นตาม**ยอดขายจริง**ที่ทีมส่งเข้า CRM — พนักงานทุกคนช่วยกัน "เลี้ยง" ตัวเดียวกันบนจอ TV ในออฟฟิศ โตเต็มวัย = ทีมรับ**รางวัลจริง** สัตว์มี**อารมณ์**ตามสถานการณ์ยอดขายสดๆ และอารมณ์มีผลต่อความเร็วการโต

- **ชื่อเกม:** Fantastic Beasts
- **จอ:** TV (web app, landscape, fullscreen, เปิดทิ้งไว้ทั้งวัน)
- **ข้อมูล:** Supabase project `crm-sale-dashboard` (`swbxecjguiohpwllvues`) — Realtime ที่ table `sales`
- **ทีม:** ล็อกเฉพาะ หนามแดง-BETA-01 (`team_id = 1`) — เป็น config เปลี่ยนทีมได้
- **ศิลป์:** Codex ปั้นภาพนิ่ง (sprite ท่าต่างๆ) + โค้ด CSS/JS ทำให้มีชีวิต (แนวทาง 1)

## 2. เฟสงาน

| เฟส | ขอบเขต | เกณฑ์ผ่าน |
|---|---|---|

## 3. กติกาเกม

### การโต
```
แต้มโต (growth points) = ยอดบาท × ตัวคูณอารมณ์ขณะยอดเข้า × ตัวคูณข้ามเดือน
```
ขั้นโต 5 ขั้น ตาม % ของเป้าตัวนั้น: **ไข่ 0% → ฟัก 10% → เด็ก 30% → วัยรุ่น 60% → เต็มวัย 100%**

### เป้า 3 ระดับ (default — ปรับได้ใน config)
| ตัว | เป้าแต้มโต | หมายเหตุ |
|---|---|---|
| ง่าย | 1,800,000 | ~70% ของยอดเดือนเฉลี่ยทีม (2–2.6 ล้าน) |
| ปานกลาง | 2,400,000 | ~ยอดเดือนเฉลี่ยพอดี |
| ยาก | 3,000,000 | ต้องทุบสถิติ — รางวัลใหญ่สุด |

เลี้ยง**ครั้งละ 1 ตัว** เริ่มจากง่าย → ปานกลาง → ยาก · 1 เดือน = 1 ตัว (เป้าหมาย)

### ระบบอารมณ์ (mood engine)
คำนวณจาก event ยอดขายจริง **เฉพาะเวลาทำงาน** (เวลางาน + วันหยุดจาก table `holidays` → config ชั่วโมงทำงานได้)

| อารมณ์ | เงื่อนไข (default) | ตัวคูณ | การแสดงออก |
|---|---|---|---|
| 🤩 ดีใจสุดๆ (thrilled) | ดีลเดียว ≥ 5,000 บาท — ค้างสถานะ 15 นาที | ×1.5 | กระโดด + พลุ particle |
| 😊 ร่าเริง (happy) | ≥3 รายการภายใน 60 นาทีล่าสุด | ×1.2 | เด้งดุ๊กดิ๊ก ตาประกาย |
| 🙂 ปกติ (calm) | สภาวะทั่วไป | ×1.0 | หายใจ กะพริบตา |
| 🥺 เหงา (lonely) | ไม่มียอด > 2 ชม. (ในเวลางาน) | ×0.8 | ตัวห่อ มองซ้ายขวา 💭 |
| 😢 เศร้า (sad) | หลัง 14:00 และยอดวันนี้ < 50% ของค่าเฉลี่ยรายวัน | ×0.7 | หัวตก น้ำตาคลอ |
| 😴 หลับ (sleep) | นอกเวลางาน / วันหยุด | — (ไม่มียอดอยู่แล้ว) | นอน zzz |

ลำดับความสำคัญเมื่อเงื่อนไขชนกัน: `sleep > thrilled > happy > sad > lonely > calm`
mood engine ต้องเป็น **pure function** `moodFor(events, now, config)` — เทสได้ deterministic (ตามแพทเทิร์นเกมที่1: state = pure actions)

### ระบบตั้งชื่อ (เพิ่ม 2026-07-11)
- ตอนสัตว์**ฟักออกจากไข่** (ข้ามจากขั้น egg) จอขึ้น overlay ให้พนักงานตั้งชื่อน้อง (พิมพ์บนเครื่องที่ต่อจอ)
- ชื่อโชว์คู่กับสัตว์ตลอด + ใช้ในป้ายข้อความ ("น้อง{ชื่อ} หิวแล้ว!") · เก็บ persist · ยังไม่ตั้ง = แสดง "???" พร้อมปุ่มตั้งชื่อ
- เปลี่ยนชื่อภายหลังได้จากปุ่มเล็กๆ (กันตั้งผิด)

### ข้ามเดือน
- จบเดือนแล้วยังไม่เต็มวัย → เลี้ยงต่อ แต่ตัวคูณโตรวม **×0.75** จนกว่าจะเต็มวัย
- เต็มวัย → ฉากฉลอง 🎉 + โชว์รางวัลจริง (ข้อความ config) → เริ่มไข่ตัวถัดไป

## 4. หน้าจอ TV (จอเดียว อยู่ยาวทั้งวัน)

```
┌──────────────────────────────────────────────┐
│  [logo Fantastic Beasts]        [อารมณ์ 😊]  │
│                                              │
│              🐉 สัตว์ตัวใหญ่กลางจอ            │
│           (idle animation ตลอดเวลา)          │
│                                              │
│  [ป้ายเด้งเมื่อยอดเข้า: รูป+ชื่อ+ยอด]           │
│ ━━━━━━━━━━ หลอดโต 42% ━━━━━━━━━━             │
│  ขั้นถัดไป: วัยรุ่น (อีก 324,000)              │
│  ticker: ยอดล่าสุด 5 รายการ · ยอดวันนี้ x บาท │
└──────────────────────────────────────────────┘
```
- ยอดเข้า → สัตว์ทำท่ารับอาหาร + ป้าย "น้อง{ชื่อ} ป้อน {ยอด} บาท!" พร้อมรูปจาก `employees.photo_url` (เฟส 2 — POC ใช้ mock)
- POC มีแผงปุ่ม dev (ซ่อนได้): ยิงยอดปลอม เล็ก/ใหญ่, เร่งเวลา, บังคับอารมณ์ — ไว้เดโม

## 5. เอกลักษณ์ภาพ (logo + UX/UI)

- **โทน:** แฟนตาซีอบอุ่นน่ารัก — เหมาะทีมหญิงล้วน · จอ TV มองไกลต้องอ่านออก (ตัวหนังสือใหญ่ contrast สูง)
- **logo "Fantastic Beasts":** ตัวอักษรโค้งมนมีเวทมนตร์ (ประกายดาว) — Codex ช่วย gen องค์ประกอบกราฟิก แต่**ข้อความไทย/อังกฤษบนภาพประกอบเองด้วยฟอนต์จริง** (บทเรียนเกมที่1: Codex gen ตัวหนังสือเพี้ยน)
- **พาเลต:** พื้นหลังค่ำคืนเวทมนตร์ (navy-violet ไล่เฉด) + สัตว์/UI สีพาสเทลสว่างตัดกัน + ทอง accent สำหรับ milestone
- **ฟอนต์:** ไทย = Kanit (คุ้นเคยจากเกมที่1) · ตัวเลข = tabular
- แอนิเมชันซ้อน sprite: หายใจ (scale), กะพริบตา (sprite สลับ), เด้ง (spring), particle (canvas/CSS)

## 6. งานศิลป์ (Codex pipeline)

- สัตว์วิเศษน่ารักแฟนตาซี (ไม่ใช่สัตว์ธรรมดา) — Codex เสนอ concept สายพันธุ์ก่อน แล้วปั้นจริง
- ภาพ **PNG/WebP โปร่งใส** ต่อสัตว์ 1 ตัว: 5 ขั้นโต × ท่า (ปกติ, ตาหลับ(กะพริบ), ดีใจ, เศร้า/เหงา, หลับ) — POC ทำเฉพาะ**ตัวง่าย** ขั้นไข่+ฟัก+เด็ก ก่อน (พอเดโมโต)
- **คุมหน้าตาตัวเดิมข้ามขั้น/ข้ามท่า:** ทุก brief แนบ reference ภาพก่อนหน้า + ระบุ "ตัวเดียวกัน สี/ลาย/เอกลักษณ์เดิม"
- ห้ามให้ Codex เขียนตัวหนังสือในภาพ · ภาพ 1024×1024 โปร่งใส แล้ว Friday แปลง/ย่อเป็น webp

## 7. สถาปัตยกรรม & โครงโค้ด

- React 18 + TypeScript + Vite + vitest (โครงเดียวกับเกมที่1) · โฟลเดอร์ `เกมที่2/app/`
- แยก layer ชัด:
  - `domain/` — pure: `growth.ts` (แต้ม/ขั้น), `mood.ts` (mood engine), `types.ts`
  - `state/` — store + config (localStorage: แต้มสะสม, ตัวปัจจุบัน — เฟส 2 จะย้าย source of truth ไปคำนวณจาก Supabase)
  - `features/tv/` — จอหลัก, ป้ายยอดเข้า, หลอดโต, ticker
  - `features/dev/` — แผง mock (POC)
  - `ui/beast/` — component สัตว์ + แอนิเมชัน (รับ props: stage, mood → เลือก sprite + motion)
- เทส: mood engine ทุกเงื่อนไข+ลำดับความสำคัญ, growth ข้ามขั้น, ตัวคูณข้ามเดือน
- Deploy: GitHub Pages (repo ใหม่ `fantastic-beasts-game`) — แพทเทิร์น Actions เดียวกับเกมที่1 · **เฟส 2 ต้องระวัง:** anon key + RLS read-only ก่อนต่อจริง (repo public)
